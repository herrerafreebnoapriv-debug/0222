# 用户列表页面说明（users.html）

本文档描述管理后台「用户列表」页面的布局、交互与样式约定。仅**超级管理员**可访问；管理员角色不显示该页面入口，直接访问会被重定向至首页。

---

## 一、查询栏布局（两行）

### 第一行
- **内容查询**：一个输入框（占位：手机号/用户名/昵称）+ 一个「内容查询」按钮。
- 输入框与按钮同一行、紧跟排列；**内容查询按钮列宽缩减约 3/4**，使用较小 padding 与字号，避免占用过宽。

### 第二行
- **[A用户名/手机]** 输入框 + **和** + **[B用户名/手机]** 输入框 + **查询用户对** 按钮。
- 「和」字位于 A、B 输入框之间，整行单独成行。
- A、B 输入框列宽各为约 187px（在 140px 基础上增加 1/3）。
- **查询用户对按钮**使用主色 **#1890FF**（悬停可用 #40a9ff）。

---

## 二、表格列

| 列名         | 说明 |
|--------------|------|
| 手机号       | 展示脱敏手机号 |
| 用户名       | 登录用户名 |
| 昵称         | 用户昵称 |
| 修改密码     | 链接，点击可对该用户发起修改密码流程 |
| 好友数量     | 数字 + 「人」+ 「查看」链接 |
| 上级人数     | 数字 + 「人」+ 「查看」链接 |
| 下级人数     | 数字 + 「人」+ 「查看」链接 |
| 注册时间     | 格式如 2025/09/09 13:59 |
| 最后上线     | 格式同上 |
| 发送管理消息 | 按钮「选择并发送」，点击打开发送管理消息弹窗 |
| 封禁         | 两个横向按钮：「禁言/解除」「永久注销」；按状态可显示已禁言/已解除/已注销 |

---

## 三、发送管理消息弹窗

- **预设内容 [1][2][3]**：每条为单选 + 可编辑输入框 + 「保存」按钮；**只能单选一条**进行发送。保存后内容可持久化（如 localStorage），再次打开弹窗时回填。
- **临时撰写**：单独一项单选 + 输入框。须**勾选该项并填写内容**后才允许发送。
- **发送按钮**：未勾选任何一项时禁用；勾选预设或勾选临时并填写内容后启用。点击发送时，若选的是临时且内容为空，应拦截并提示。
- 弹窗内按钮样式与主色一致（#1890FF 等）由全局或弹窗内样式统一。

---

## 四、查询用户对弹窗

- 展示**关系**、**初次聊天时间**。
- **聊天内容搜索**：输入框 + **搜索/查询**按钮；输入框宽度较大（如 flex:1、min-width:200px），**查询按钮为小号**，不占过大空间。
- **导出聊天记录**按钮：小号按钮，主色 **#1890FF**。
- 弹窗内所有操作按钮（查询、导出聊天记录）统一使用 **#1890FF**。

---

## 五、底部说明文案

页面底部保留三段说明（可折叠或常显）：

1. **封禁**：[禁言/解除][永久注销] 为两个横向排列按钮，根据状态显示已禁言/已解除、已注销。
2. **发送管理消息**：按钮点开后弹窗内展示 [1][2][3] 预设内容（可编辑和保存）、[4] 临时撰写输入框，勾选后点击发送向目标发送系统消息。
3. **查询用户对**：弹窗显示两用户关系、初次聊天时间、聊天内容搜索框及两人聊天记录导出按钮。

---

## 六、样式与多语言

- 主色 **#1890FF** 用于：查询用户对按钮、查询用户对弹窗内的查询按钮与导出聊天记录按钮。
- 所有文案通过 i18n 配置（`assets/js/i18n.js`），支持中/英切换；键包括但不限于：contentQueryBtn、queryUserPairBtn、inputAPlaceholder、inputBPlaceholder、andLabel、changePasswordCol、friendsCount、superiorsCount、subordinatesCount、regTime、lastOnline、sendMgmtMsg、selectAndSend、ban、muteUnmute、permanentLogout、userPairModalTitle、mgmtMsgModalTitle、exportAllChats、saveBtn、savedTip 等。
