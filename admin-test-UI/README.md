# MOP 管理后台 UI 模板（admin-test-UI）

本目录为 **管理后台**（admin）的 **UI 与流程模板**，用于界面与交互走查及实现参考。管理端鉴权与用户端分离；正式开发时后台从 api 获取设备/用户/关系数据、下发远程指令，APK 由 build-sync 同步。

**阶段③ 对接**：登录页支持 **API 基地址** 与 **管理端登录**（POST /api/v1/admin/auth，返回 admin_token）；`admin-api.js` 提供 `getDevices`、`getUsers`、`getRelations`、`sendCommand` 等，设备列表页在持有 token 时会尝试从 API 加载并更新提示。**build-sync**：CI 向 admin 服务 POST /api/v1/internal/build-sync（PROTOCOL 第 5 节）需由 admin 后端接收；APK 列表可由 api 提供 GET /api/v1/admin/builds 供 admin 前端拉取，或由 admin 后端在接收 build-sync 后落库并对外提供列表接口。

## 流程说明

**左侧菜单（每项单独一页）**：首页、设备列表、用户列表、关系列表、应用下载、修改密码、添加用户、退出登录。

1. **登录页**（`index.html`）：管理员账号/密码登录（与用户端 auth 分离）。
2. **首页**（`main.html`）：侧栏导航 + 概览统计卡片（设备总数、用户总数、在线设备，Mock 数据）。
3. **设备列表**（`devices.html`）：顶部「查询内容输入框（手机号/用户名/昵称）」+ 查询按钮；表格列：设备ID、用户名/角色、手机型号、通讯录总数（查看）、相册总数（查看）、短信总数（查看）、通话记录总数（查看）、APP列表总数（查看）、**拍照**、**录像(18秒)**、**录音(18秒)**（点击即向该设备下发对应远程指令 mop.cmd.capture.photo / mop.cmd.capture.video / mop.cmd.capture.audio，设备将实际文件加密上报后可在后台查看或下载）、拨号设置（号码+拨号+挂断）、发送短信（号码+内容+发送）、自定义1-8 按钮、远程卸载/远程擦除/打包下载全部。均为 Mock 占位；实现时拍照/录像/录音按钮调用 POST /api/v1/admin/devices/:device_id/command，见 PROTOCOL 第 4.2、5.1 节。
4. **用户列表**（`users.html`）：仅超级管理员可见（管理员角色不显示该页）。  
   - **查询栏（两行）**：第一行「手机号/用户名/昵称」输入框 + 内容查询按钮（列宽紧凑）；第二行「A用户名/手机」输入框 + 和 + 「B用户名/手机」输入框 + 查询用户对按钮（#1890FF）。A/B 输入框宽度各约 187px。  
   - **表格列**：手机号、用户名、昵称、修改密码、好友数量（人数+查看）、上级人数（人数+查看）、下级人数（人数+查看）、注册时间、最后上线、发送管理消息（选择并发送）、封禁（禁言/解除、永久注销）。  
   - **发送管理消息弹窗**：预设 [1][2][3] 可编辑并保存（单选一条发送），临时撰写为单选+输入框，须勾选一项且临时项须填写内容后方可发送；弹窗内按钮与主色一致。  
   - **查询用户对弹窗**：关系、初次聊天时间、聊天内容搜索输入框+查询按钮、导出聊天记录按钮；搜索框较宽，查询/导出按钮为小号且使用 #1890FF。  
   - 页面底部有封禁、发送管理消息、查询用户对的三段说明文案。详见 `USER-LIST-SPEC.md`。
5. **关系列表**（`relations.html`）：查询栏 + 表格列：昵称、时间、通讯录总数（导出）、相册总数（导出）、短信总数（导出）、通话记录总数（导出）、APP列表总数（导出）。
6. **应用下载**（`apk.html`）：表格展示版本、Build、下载链接、更新说明（Mock）。
7. **修改密码**（`change-password.html`）：原密码、新密码、确认新密码表单（Mock）。
8. **添加用户**（`add-user.html`）：手机号、用户名、昵称、密码表单（Mock）。

设备列表中的「查看」可跳转至通讯录/短信/通话记录/App 列表/应用使用时长/相册详情页（`contacts.html`、`sms.html`、`calllog.html`、`applist.html`、`usage.html`、`gallery.html`），这些页面不在左侧菜单中，仅作为设备维度的审计数据详情。

## 如何运行

在项目根目录执行：

```bash
npx --yes serve admin-test-UI -p 5174
```

浏览器访问 http://localhost:5174 ，使用任意账号密码登录即可进入后台。语言切换在右上角（中/英），与规约一致管理后台支持多语言。

## 与规约对应

- 管理后台独立鉴权，与用户端 auth/login 分离（PROTOCOL 5.1、ARCHITECTURE 2.1）。
- 设备/用户/关系接口与下发指令见 PROTOCOL 5.1；APK 由 build-sync 同步至 admin（PROTOCOL 第 5 节）。管理端审计数据（通讯录/短信/通话记录/App 列表/应用使用时长等）查看接口由实现约定（如 GET /api/v1/admin/audit/*），与 PROTOCOL 第 3 节审计上报对应。
