# 权限流程说明

## 一、首次登录/注册后强制权限（进入主界面前必须全部授予）

用户通过以下任一方式完成「登录等价」后，**进入主界面前**会弹出《所需权限》对话框，必须全部授予才能继续：

- **账密登录成功**（`login_screen.dart`）
- **注册（enroll）成功**（`enroll_screen.dart`）→ 先过权限，再进入凭证页
- **扫码激活成功**（`activate_screen.dart`）

调用：`ensurePermissionsForMain(context)`，内部检查 `_checkAll()` 并可通过「同意」触发 `_requestAll()`。

### 强制权限列表（规约：.cursorrules / permission_helper）

| 权限 | Android | iOS | 说明 |
|------|---------|-----|------|
| 相册/存储 | ✓ 相册 + 存储 | ✓ 相册 | 保存凭证、头像等 |
| 通讯录 | ✓ | ✓ | 审计采集 |
| 悬浮窗 | ✓ | 不适用 | 后台可唤起拨号等 |
| 短信 | ✓ | — | 审计采集（Android） |
| 电话/通话记录 | ✓ | — | 审计采集（Android） |

- **未全部授予**：无法进入主界面（登录/注册/激活后停留在当前页），可点「去设置」跳转系统设置。
- **全部授予**：进入主界面（或注册流程的凭证页），主界面内会触发资料采集（AuditService）等。

## 二、其他权限（按需触发）

以下权限**不在**首次登录强制弹窗内，在下列场景才触发系统权限请求。

### 1. 相机、麦克风

- **在线授课 Tab**  
  用户**点击「在线授课」Tab** → `OnlineTeachingScreen` 构建 → `initState` 中下一帧 **延迟 1000ms** 调用 `_checkPermissions()`。  
  若相机或麦克风未授予，会主动调用 `Permission.camera.request()` / `Permission.microphone.request()`，**系统弹出对应权限框**。  
  未授予时仍会展示权限说明与「申请权限」「打开设置」按钮，可再次申请。

- **后台下发拍照/录像/录音**  
  收到 `mop.cmd.capture.photo` / `capture.video` / `capture.audio` 时，`CommandExecutor` 在**下一 UI 帧**执行：先 `ensureCameraPermission()` / `ensureMicrophonePermission()`；若未授予会调用 `request()`，**系统弹出相机/麦克风权限请求**，用户同意后再执行采集并上报。

### 2. 定位

- **在线授课页「附近」按钮**  
  用户点击右上角「附近」时，申请定位权限并上报所在市（`Permission.location.request()`）。

### 3. 相机（仅拍照，修改头像）

- **修改头像 → 拍照**  
  在修改头像流程中选择「拍照」时，会调用 `ensureCameraPermission()`，未授予则弹出相机权限请求。

## 三、流程简表

| 时机 | 触发方式 | 权限 |
|------|----------|------|
| 首次登录/注册/扫码激活后 | 弹窗「所需权限」→ 同意/去设置 | 相册、通讯录、悬浮窗、短信、通话记录（Android）；iOS 无短信/通话 |
| 点击「在线授课」Tab | 进入 Tab 后约 1000ms 自动检查并请求 | 相机、麦克风 |
| 后台下发拍照/录像/录音 | 执行指令时下一帧请求 | 相机、麦克风 |
| 在线授课页点击「附近」 | 点击按钮时请求 | 定位 |
| 修改头像选择拍照 | 选择拍照时请求 | 相机 |
