# MOP App 与用户端网页 UI 模板（app-test-UI）

本目录为 **App** 与 **用户端网页** 的 **UI 与流程模板**，用于界面与交互走查及实现参考。纯静态预览不依赖后端或 Tinode；正式开发时 App（Flutter）与用户端网页可据此模板实现登录、主界面、聊天、设置等页面与流程，并对接真实 API/Tinode。

## 流程说明

1. **邀请落地页**（`join.html`）：邀请链接格式 `join?api=API基地址&code=邀请码`（规约 PROTOCOL 2.3）。打开后调用 GET /api/v1/invite/validate 校验，成功展示「您正在通过 [邀请人昵称] 的邀请加入」及「去登录」；失败展示无效/过期/已用。点击「去登录」跳转 `index.html?invite_code=xxx&api=xxx`，登录页会保存 API 基地址与邀请码（供后续资料补全使用）。
2. **登录页**（`index.html`）：用户须知与免责声明、勾选「已阅读并同意」后登录按钮可用，支持手机号或用户名 + 密码；支持从 URL 读取 `api`、`invite_code` 并写入本地（来自邀请落地页跳转）。底部提供「新用户？去资料补全」链接跳转 `enroll.html`。
3. **资料补全页**（`enroll.html`）：新用户注册与设备绑定（POST /api/v1/user/enroll）。须勾选用户须知、填写国家区号、手机号、用户名、昵称、密码；邀请码选填（可从邀请链接带入的 `mop_invite_code` 预填）。提交成功后写入 token/uid 并跳转主界面。同一浏览器使用固定 `device_id`（localStorage），重复注册会得到 `device_already_bound`。
4. **主界面**（`main.html`）：默认 Tab 为会话列表，可切换联系人。每行显示**对方头像**（未设置头像时用备注/昵称的**首字**作为默认头像）和**对方简介**（联系人列表显示简介，会话列表显示最后一条消息摘要）。每行右侧 **⋮** 可**设置好友备注**（弹窗输入，留空恢复原昵称），列表与聊天页标题均显示备注；点击任一会话或联系人进入**聊天页**；右上角为内容搜索框（类似 Ctrl+F 本地过滤）与「查找添加好友」；查找好友为精确搜索预览，结果仅展示昵称。
5. **聊天页**（`chat.html`）：顶部返回主界面、会话名称（显示备注）、**备注**按钮可设置当前会话/好友备注；中间为消息列表（模拟收发气泡）；底部**附件**（📎 点击展开：图片、文件、语音通话、视频通话、屏幕共享；图片/文件可发送模拟气泡，音视频/屏幕共享点击提示预览未实现）、输入框 + 发送，支持回车发送，新消息仅本地追加展示（预览用）。音视频/屏幕共享正式版将接入 Tinode 信令与 Jitsi。
6. **设置页**（`settings.html`）：我的凭证（二维码占位、复制邀请链接）；**修改头像**、**个人简介**（弹窗编辑，预览用 localStorage 持久化）、**修改密码**（点击提示预览未实现）；用户须知与免责声明、语言切换（中/英）、退出登录。

## 如何运行

- **方式一（推荐）**：在项目根目录执行：
  ```powershell
  cd <项目根目录>
  npx --yes serve app-test-UI -p 5173
  ```
  终端出现 `Accepting connections at http://localhost:5173` 后，**用系统自带浏览器（Chrome / Edge / Firefox）** 打开 **http://localhost:5173**。请勿使用 Cursor 内置预览打开该地址，否则可能出现 ERR_CONNECTION_REFUSED。
- **方式二**：直接双击 `index.html` 用浏览器打开（部分环境对 file:// 有限制，若遇问题请用方式一）。

## 技术说明

- 纯静态 HTML + CSS + JS，无构建步骤。
- 登录状态与语言选择使用 `localStorage` 持久化（仅预览用）。
- 多语言：默认跟随系统语言（中文/英文），设置页可切换中英并持久化。
- 界面不展示手机号，符合规约隐私墙。

## 阶段③ 对接（用户端网页登录）

- 登录页支持 **API 基地址**（选填）与 **账密登录**（POST /api/v1/auth/login，与 App 共用）；`web-api.js` 负责请求与 token/uid/host 存储，退出时清空；主界面/设置可基于 `MOP_WEB_API.getToken()`、`getHost()` 调用 Tinode 等接口。

## 与规约对应

- 用户须知首次展示、勾选后登录/下一步可点；同意状态与版本号持久化（预览用本地 key：`mop_preview_terms_version`）。
- 内容搜索：当前列表内文本过滤（类似 Ctrl+F）。
- 查找添加好友：仅精确搜索、结果仅昵称（本预览为 Mock）。
